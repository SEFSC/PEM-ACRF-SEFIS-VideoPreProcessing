Read the Docs

Python utility for clipping and splicing GoPro video from SEFIS video surveys

This is a simple Python utility used for merging GoPro videos collected during SEFIS video surveys. GoPro cameras are deployed for about an hour at a time and record continuously during this period, but the video files are automatically split into 4GB chunks by GoPro video processing software, which each file containing about 15 minutes of footage. Current protocol involves processing a 24-minute video segment from each deployment relative to the time the camera reaches the bottom.

About

This script does three things:

  1. Determines where in the collection of videos for a given deployment the desired 24-minute segment is located using a csv file containing timestamps for when the camera reached the bottom
  2. Clips the relevant portion of partial videos at the appropriate start and end times
  3. Stitches the required videos together into a single video file for each deployment.

An optional diagnostic mode can be used to embed the video timestamp into the video frames (formatted HH:MM:SS:FF) to help find the splice times, since some video software do not preserve the original frame rate or fail to display the full time during playback. Engaging diagnostic mode will also print out a table of the splice times for each deployment to the console including where in the original video files the segments were taken.

Note

This table is always added to a log file even when diagnostic mode is off, but printing to the screen can be helpful for troubleshooting and verifying the splice times.

Usage

Running the script is straightforward. After activating the appropriate Python environment, simply call the script from the command line specifying the configuration YAML file to use:

python clip_and_splice.py configurations.yaml

If the configuration file is not co-located with the script, simply include the path to the file so that the script knows where to find it:

python clip_and_splice.py path/to/configurations.yaml

for more information, including an execution example, see Using the Model.

Options

All user options are contained in a config YAML file (called configurations.yml by default, but can be named anything) to allow easier control and greater reproducibility. Settings are entered as key: value pairs as described below.

Paths and associated settings
Key Description
input_directory Path to parent folder containing subfolders for set of videos. The subfolders should be named according to the deployment name (e.g., DEPLOYMENT_001), which will be used to name the new video file, and should contain all of the video files for that deployment.
output_directory Path to folder where the new video file will be saved. The new video will automatically be named according to the deployment name (e.g., DEPLOYMENT_001.MP4).
csv_path Path to the “time-on-bottom” CSV file. This file should contain entries for each deployment (subfolder) in rows with deployment name in one column and “time-on-bottom” in a separate column. Any other columns will be ignored.
col_foldername Column name for the column containing the deployment or folder name. Defaults to foldername but can be changed here if needed.
col_timebottom Column name for the column containing the “time-on-bottom” timestamps. Defaults to time_on_bottom but can be changed here if needed.
Processing settings
Key Description
reprocess If true, existing stitched videos will be overwritten. Otherwise, skips folders where the output file already exists. Defaults to false.
diagnostic_mode If true, runs in diagnostic mode which prints the video cut times and embeds the time into the video itself, which is independent of the video player’s time stamp, to verify splicing. Defaults to false.
use_gpu Whether to use GPU acceleration, if available. Requires GPU hardware and compatible ffmpeg build (see below). Defaults to false.
Video processing utilities. These should not need to be changed once set but may vary between computers and users.
Key Description
ffmpeg_path Path to the executable ffmpeg utility (see below).
ffprobe_path Path to the executable ffprobe utility (see below).
ImportantA note on file paths

Use forward slashes (/) or double backslashes (\) for Windows paths to avoid issues with escape characters. For example, C:/Users/username/Documents/configurations.yaml or C:\\Users\\username\\Documents\\configurations.yaml.

Dependencies

  • Git CLI (for repository cloning)
  • Python >= 3.14
  • ffmpeg and ffprobe >= 6.1

Both ffmpeg and ffprobe utilities are included in the ffmpeg download available here.

Python Libraries

  • pandas >= 3.0.2
  • pyyaml >= 6.0.3
  • tqdm >= 4.67.3

These can be installed from the accompanying `requirements.txt’ file. See Getting Started for instructions on how to set up a Python environment and install dependencies.

Recommendations

  • Anaconda (for virtual environment implementation)
